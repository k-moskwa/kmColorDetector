typedef void SwtCallback(void *, SwtValueType *)

Definition of the Software Timer Callback.

Parameters
Pointerfor void content that is registered in swtRegisterCallback function and will be delivered to callback.
SwtValueTypeNeed to be altered to the next interval for the Software Timer. If not changed, this will be the last call of specific software timer, and the next time it needs to be reissued with swtStart. This can be useful to define "single shot" type of software timer.

◆ swtInit()

void swtInit ( int16_t  milisecondsResolution)

Initialization of the Software Timer routines with main .

NOTE: Although even seconds are possible, for typical applications and quartz values best option is to use 1ms. Following definitions to be set in config.h file
#define SWT_SIZE_OF \ Number of available software timers. To be adjusted to the needs.

#include <stdbool.h>
#include <stdint.h>
#include <avr/io.h>
#include <avr/interrupt.h>
void callbackDebugLed(void *userData, SwtValueType *newTimerValue) {
PORTC ^= 0x80;
*newTimerValue = 500;
}
int main(void)
{
DDRC |= 0x80;
PORTC &= ~0x80;
swtInit(SWT_INTERVAL_1MS);
swtRegisterCallback(SWT_TIMER_0, 0, callbackDebugLed);
swtStart(SWT_TIMER_0, 500);
sei();
while (true) {
swtLoop();
continue;
}
Parameters
milisecondsResolutionMain interval of the software timers defined in miliseconds.

Timer2 mode 2 - CTC top value OCR2

Timer/Counter2 Output Compare Match Interrupt Enable
